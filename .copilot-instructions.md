# LivingPatch MVP — Copilot Instructions

**Quick start for agents**: Read `context.md` first (2 min quick reference). Then use this file for detailed tech stack and data model guidance.

## Project Overview
LivingPatch is a personal ecological literacy tool — a local webapp for exploring NE Pennsylvania species and their ecological relationships. **MVP (Phase 1)**: read-only dataset explorer. No server, no accounts, no external APIs. Fully offline.

## Tech Stack
- **Framework**: React 18 + TypeScript (strict mode)
- **Build**: Vite 8
- **Styling**: TailwindCSS v4 with `@tailwindcss/vite` plugin
- **Routing**: React Router v6 (HashRouter for offline use)
- **State**: React hooks only (Phase 1); Zustand for Phase 2
- **Data**: Static JSON dataset indexed into Maps for O(1) lookups

## Project Structure
```
app/src/
  types/
    index.ts          → Species, Symbiosis, Relation, LifeStage interfaces
  data/
    dataset.json      → Static species database (~30 species + 8 groups, 50+ relationships)
    index.ts          → Load JSON + build lookup maps (speciesById, symbiosisBySpeciesId, relationsBySpeciesId)
  hooks/
    useDataset.ts     → Expose data module as React hook
  lib/
    filters.ts        → filterSpecies(species, filters) + getFilterOptions()
    relationships.ts  → getRelatedEntries(), groupByRole() for symbiosis grouping
    labels.ts         → Human-readable labels for all tag types (form, season, habitat, diet, behavior, symbiosis)
  components/
    SearchBar.tsx     → Debounced text search input
    FilterPanel.tsx   → Chip-style form/season/habitat multi-select filters
    SpeciesList.tsx   → Species list rows + species groups (lighter styling)
    SpeciesCard.tsx   → Full detail view: name, tags, description, keystone, life stages, relationships
    RelationshipsPanel.tsx  → Sections grouped by symbiosis type; obligate pinned; max 6 per section
    RelationshipTile.tsx    → Tappable link to related species (or non-navigable if group)
    LifeStageRow.tsx  → Grid of life stage icons + name + active months
    KeystoneBadge.tsx → Styled badge for keystone species + type
  pages/
    HomePage.tsx      → SearchBar + FilterPanel + SpeciesList
    DetailPage.tsx    → Load species by `:id` param, render SpeciesCard
  App.tsx             → HashRouter, routes, sticky header
  main.tsx            → ReactDOM render
  index.css           → @import "tailwindcss" + global resets
```

## Data Model
### Species
- `id`: type_common-name-slug (e.g. `bird_pileated-woodpecker`)
- `common_name`, `latin_name`, `form`, `habitat[]`, `diet[]`, `behavior[]`, `season[]`
- `functional_description`: plain English, ecological role
- `life_stages[]`: { icon, name, description, months[] }
- `is_keystone: boolean`, `keystone_type`, `keystone_description`
- `is_group: boolean` (groups shown with lighter styling, non-navigable)

### Symbiosis
- `type`: mutualism | parasitism | predation | competition | commensalism
- `members[]`: [speciesIdA, speciesIdB]
- `impacted_species?`: benefiting/harmed species ID (parasitism, predation, commensalism)
- `obligate: boolean` — pinned at top of section if true
- `notes`: relationship context

### Relations
- `type`: string (e.g., "taxonomic_group")
- `members[]`: species IDs related
- `notes`: context

## Key Conventions
- **Components**: PascalCase, one component per file in `src/components/`
- **Hooks**: `useXxx` pattern in `src/hooks/`
- **Utilities**: camelCase in `src/lib/`
- **Types**: All exported from `src/types/index.ts`
- **Styling**: Tailwind v4 classes; color palette: stone (neutrals), emerald (primary), amber (accents/keystone)
- **Search**: Full-text across common_name, latin_name, functional_description, all tags
- **Filters**: Form, season, habitat (multi-select, combinable with search)

## Critical Details
1. **Relationships panel**: Always group by symbiosis type. Obligate relationships MUST be pinned to top of section.
2. **Groups** (`is_group: true`): Show in list with lighter styling, non-clickable. Do NOT render detail page.
3. **HashRouter**: Use for offline-first capability; no server dependency ever.
4. **No external APIs**: Zero network calls. All data bundled at build time.
5. **Life stages**: Only render if `life_stages` array contains objects with `icon`, `name`, `months`.


## Product Principles
- **Relationships first** — every species shown in ecological context
- **No identification** — users identify externally (Merlin, iNaturalist); LivingPatch logs + explores
- **No social** — fully private, local-only
- **Functional, not taxonomic** — describe species by what they *do*, not Latin names
- **Quiet** — minimal UI, clean design, focus on data

## When Adding Features
1. **Search logic**: Update `lib/filters.ts` `filterSpecies()` 
2. **New relationship type**: Update `SymbiosisRole` in `lib/relationships.ts` + `symbiosisLabel()` in `lib/labels.ts`
3. **New filter**: Add option to `FilterPanel.tsx` + toggle logic + label utility
4. **Life stage display**: Verify `life_stages` shape in data; update `LifeStageRow.tsx` if needed
5. **Styling**: Use Tailwind classes; extend in `index.css` if needed (rare)
