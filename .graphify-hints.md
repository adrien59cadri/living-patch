# Graphify Hints: Living-Patch Data Model & Architecture

This file helps graphify understand the semantic relationships in the LivingPatch codebase, particularly the ecological data model and how the app consumes it.

## Core Data Model

### Species
Located in `app/src/data/dataset.json` (generated from `pack-tools/packs/`):
- **Individual species**: ~30 species with ecological attributes
  - `id`: Unique identifier (e.g., "plant_common-milkweed")
  - `common_name` & `latin_name`: Scientific naming
  - `form`: Category (bird, plant, butterfly, insect, amphibian, etc.)
  - `habitat[]`, `diet[]`, `behavior[]`, `season[]`: Tag arrays for filtering
  - `functional_description`: Natural language description of ecological role
  - `life_stages[]`: Growth/metamorphosis stages
  - `active_months`: Seasonal activity (1-12)
  - `is_keystone`: Boolean flag for keystone species
  - `keystone_type`: Type of keystone role (engineer, predator, mutualist, etc.)
  - `keystone_description`: Explanation of keystone importance

- **Taxonomic groups**: ~8 groups
  - `is_group: true` flag
  - `members[]`: Array of species IDs in the group

### Symbiosis
~50 relationships between species representing ecological interactions:
- `type`: Relationship category (mutualism, parasitism, predation, competition, commensalism)
- `members[]`: Array of 2+ species IDs involved
- `obligate`: Boolean—true if the relationship is required for survival
- `notes`: Description of the interaction

### Relations
~8 entries defining taxonomic and ecological groupings:
- `type`: Group type (e.g., "form" for birds, plants, insects)
- `members[]`: Species IDs in the relation
- `notes`: Explanation of grouping

## Component Data Flows

### Search & Filter System
- **SearchBar** (`app/src/components/SearchBar.tsx`): Full-text search input
- **FilterPanel** (`app/src/components/FilterPanel.tsx`): Multi-select filters for form, season, habitat
- **useDataset hook** (`app/src/hooks/useDataset.ts`): Loads and indexes dataset
- **filters.ts** (`app/src/lib/filters.ts`): Core filtering logic
  - `getUniqueFormTags()`: Extract all forms from species
  - `getUniqueSeasonTags()`, `getUniqueHabitatTags()`: Same for other dimensions
  - `filterSpecies()`: Main filter function (combines search + all filter dimensions)
  - Returns filtered species list to SpeciesList

**Data flow**: SearchBar + FilterPanel → useDataset + filters.ts → SpeciesList display

### Species Detail & Relationships
- **DetailPage** (`app/src/pages/DetailPage.tsx`): Main detail view for a single species
- **SymbiosisSection** (`app/src/components/SymbiosisSection.tsx`): Displays obligate and other relationships
- **relationships.ts** (`app/src/lib/relationships.ts`): Core relationship queries
  - `getSymbiosisBySpeciesId()`: O(1) lookup of symbiosis entries for a species
  - `groupSymbiosisByType()`: Groups relationships by type (mutualism, parasitism, etc.)
  - `getSorted()`: Sorts obligate entries to top
  - `getRelationshipLabel()`: Human-readable labels
- **RelationshipTile** (`app/src/components/RelationshipTile.tsx`): Individual relationship card

**Data flow**: DetailPage loads species → queries relationships.ts → renders SymbiosisSection → user can click related species

### Navigation & Neighbors
- **NeighborsGrid** (`app/src/components/NeighborsGrid.tsx`): Category-based drill-down
- **NeighborListView** (`app/src/pages/NeighborListView.tsx`): Full list of related species by category
- **TaxonomyRelatedGrid** (`app/src/components/TaxonomyRelatedGrid.tsx`): Taxonomic group connections
- **Router** (`app/src/App.tsx`): Handles navigation between DetailPage → NeighborListView → DetailPage

**Data flow**: DetailPage → click NeighborsGrid category → NeighborListView loads all species in that category → click species → DetailPage with new species

## Data Pack System

Located in `pack-tools/`:
- **packs/** directory: Individual JSON files (0-base.json published, examples as drafts)
- **lib/schema.ts**: Zod validation schemas for Species, Symbiosis, Relations
- **lib/merge.ts**: Merges multiple packs into single dataset
- **lib/conflicts.ts**: Detects conflicts when merging packs
- **cli/validate.ts**: Validates pack structure against schema
- **cli/fetch-images.ts**: Fetches Wikipedia images for species in packs
- **Build process**: `build-dataset.js` (root) → merges packs → generates `app/src/data/dataset.json`

**Key insight**: Packs are modular, versioned data containers. The app loads the merged result from dataset.json, indexed into Maps for O(1) lookups.

## Type System

**Central type definitions** in `app/src/types/index.ts`:
- `Species`: Full species definition with all fields
- `SpeciesGroup`: Taxonomic group definition
- `Symbiosis`: Relationship between species
- `Relation`: Grouping of species
- `DatasetIndex`: Indexing structure (Maps for fast lookup)
  - `speciesById`: Map<id, Species>
  - `symbiosisBySpeciesId`: Map<id, Symbiosis[]>
  - `relationsByType`: Map<type, Relation>

**All components import from here**. Type safety is strict (TypeScript strict mode).

## Key Architectural Patterns

1. **O(1) Lookups**: All data is indexed into Maps during initialization
   - `speciesById` for species lookup
   - `symbiosisBySpeciesId` for relationship queries
   - Avoids O(n) array searches in hot paths

2. **Immutable Data**: Dataset is read-only in Phase 1 (MVP)
   - Phase 2 will add localStorage for user sightings

3. **Memoization**: Expensive computations cached with useMemo
   - FilteredSpecies list in FilterPanel
   - Grouped relationships in SymbiosisSection
   - Neighbor categories in NeighborsGrid

4. **Mobile-First CSS**: Tailwind v4 with responsive design
   - `sm:` breakpoint for tablets
   - `lg:` breakpoint for desktop
