## Common Patterns to Reference

### Adding a component
Ask Claude/Copilot to create a new component in `src/components/` following the pattern of existing ones (e.g., RelationshipTile). Include Props interface, Tailwind styling, and clear exports.

### Modifying filters
Changes usually go in `src/lib/filters.ts` (filterSpecies function, getFilterOptions) and `src/components/FilterPanel.tsx` (UI). Update label helpers in `src/lib/labels.ts` if adding new tag types.

### Debugging relationships
Check `lib/relationships.ts` (getRelatedEntries, groupByRole) and verify symbiosis data in the JSON dataset. Use browser DevTools to inspect Map objects: `window.__data` or similar if exported.

### Styling issues
Look at existing components (e.g., SpeciesCard, SpeciesList) for TailwindCSS patterns. Color palette: stone (neutrals), emerald (primary), amber (keystone). Breakpoints: `sm:`, `lg:`.

### Type errors
All types live in `src/types/index.ts`. Import Species, Symbiosis, etc. from there. Use `as const` for literal unions. Avoid `any`.

## Before You Ask

1. Check if similar code already exists in the codebase (search SpeciesList for filter pattern, RelationshipsPanel for grouping, etc.)
2. Review MEMORY.md for conventions (component names, file structure, styling approach)
3. Verify the dataset structure if working with data (check app/src/data/dataset.json)
4. Clear your browser cache/hard refresh if changes don't appear
5. Run `npm run dev` and check console for errors

---

## After You Get Help

1. Run the dev server (`npm run dev`)
2. Test the change in browser (search, filter, navigate, go offline)
3. Check TypeScript (`npx tsc --noEmit` from app/)
4. Commit with clear message: `git commit -m "feat: [brief description]"`
5. Update TODO.md if task was tracked
